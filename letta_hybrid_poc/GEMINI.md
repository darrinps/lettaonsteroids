# Gemini CLI Project

This is a starter `GEMINI.md` file for your project. You can use this file to document anything relevant to your interactions with the Gemini CLI, such as:

- Project setup notes
- Commands used
- Key decisions or strategies
- Code snippets or configurations
- Testing procedures


### ðŸŽ¯ Things You Can Customize

The file is a simple Markdown file, and you can use it to define any of the following:

##  * **Persona:** You can tell Gemini *who* it should be.
##      * *Example:* "You are an expert Go developer."
##  * **Tone & Style:** You can define *how* it should respond.
##      * *Example:* "Be direct and concise. Do not use conversational filler."
##  * **Rules & Constraints:** You can set strict rules for its output.
##      * *Example:* "Do not use apologies or subjective phrases like 'I think'."
##  * **Output Format:** This is very powerful. You can force it to respond in a specific structure.
##      * *Example:* "All code must be in fenced code blocks with the language specified."
##  * **Background Context:** You can provide information about your project or yourself that it should "remember."
##      * *Example:* "I am working on a project using Python 3.11 and the Flask framework."
